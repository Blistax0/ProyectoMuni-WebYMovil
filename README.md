## 2. Descripción de la Entrega (EP 1)
Este repositorio contiene el avance correspondiente a la **Entrega Parcial 1: Diseño y Estructura inicial**. El objetivo es definir el diseño de la aplicación y construir la estructura base del frontend utilizando **Ionic + React**.

### Requerimientos Funcionales (EP 1.1)
Se han definido 7 requerimientos funcionales que cubren los roles de **Administrador** y **Patrullero**:
1. **Registro de Incidentes Georreferenciados:** Captura multimedia en terreno.
2. **Monitoreo de Flota:** Visualización en tiempo real sobre mapa interactivo.
3. **Gestión de Geo-cercas:** Definición de perímetros virtuales de vigilancia.
4. **Alertas Operativas:** Notificaciones por exceso de velocidad o salida de perímetro.
5. **Historial de Rutas:** Reproducción de recorridos históricos de las unidades.
6. **Botón de Emergencia (Pánico):** Alerta prioritaria con ubicación exacta.
7. **Trazabilidad de Casos:** Derivación y cambio de estado de incidentes.

### Requerimientos No Funcionales (EP 1.1)
1. **Rendimiento:** Latencia de actualización GPS menor a 2 segundos.
2. **Seguridad:** Cifrado de datos y autenticación mediante JWT.
3. **Usabilidad:** Interfaz de alto contraste para visibilidad en exteriores.

## 3. Justificación del Problema (EP 1.2)
El proyecto aborda el déficit de vigilancia y la baja respuesta policial en sectores residenciales y costeros. Al digitalizar la gestión (antes manual), se elimina la pérdida de información y se permite una coordinación institucional más fluida.

## 4. Tecnologías Utilizadas
* **Frontend:** Ionic Framework con React y TypeScript.
* **Navegación:** React Router para rutas públicas y protegidas.
* **Gestión:** GitHub Projects e Issues.
